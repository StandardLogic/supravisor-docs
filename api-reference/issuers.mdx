# Issuers API

Issuers are the trust anchors in Supravisor. Every passport must be signed by an issuer, and gates verify passports by checking whether the signing issuer is trusted.

**Base URL:** `https://uniplex.ai/api`

---

## List Issuers

Retrieve all issuers for the authenticated account.

```
GET /api/issuers
```

**Required scope:** `issuers:read`

**Query Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `status` | string | Filter by `active` or `revoked` |
| `limit` | integer | Results per page (default: 50, max: 200) |
| `offset` | integer | Pagination offset |

**Example Request:**

```bash
curl https://uniplex.ai/api/issuers \
  -H "Authorization: Bearer uni_xxxx"
```

**Response:**

```json
{
  "issuers": [
    {
      "issuer_id": "issuer:acmecorp.ai",
      "name": "Production Agents",
      "description": "Signs all production agent credentials",
      "trust_tier": "internal",
      "public_key": "base64_ed25519_public_key...",
      "passport_count": 12,
      "status": "active",
      "created_at": "2026-01-15T00:00:00Z",
      "updated_at": "2026-01-15T00:00:00Z"
    }
  ],
  "total": 3,
  "limit": 50,
  "offset": 0
}
```

## Create Issuer

Create a new issuer. The Ed25519 keypair is generated server-side for API calls; use the dashboard for browser-side key generation.

```
POST /api/issuers
```

**Required scope:** `issuers:create`

**Request Body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | ✅ | Human-readable name |
| `issuer_id` | string | ✅ | Unique identifier in `issuer:domain` format |
| `description` | string | ❌ | Optional notes |
| `public_key` | string | ❌ | Base64 Ed25519 public key (if you generated the keypair yourself) |

**Example Request:**

```bash
curl -X POST https://uniplex.ai/api/issuers \
  -H "Authorization: Bearer uni_xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Production Agents",
    "issuer_id": "issuer:acmecorp.ai",
    "description": "Signs all production agent credentials"
  }'
```

**Response:**

```json
{
  "issuer_id": "issuer:acmecorp.ai",
  "name": "Production Agents",
  "trust_tier": "internal",
  "public_key": "base64_ed25519_public_key...",
  "private_key": "base64_ed25519_private_key_SHOWN_ONCE",
  "status": "active",
  "created_at": "2026-02-24T10:00:00Z"
}
```

<Warning>
  The `private_key` field is only included in the creation response and is never returned again. Store it immediately in your secrets manager.
</Warning>

## Get Issuer

Retrieve a single issuer by ID.

```
GET /api/issuers/:issuer_id
```

**Required scope:** `issuers:read`

**Example Request:**

```bash

  -H "Authorization: Bearer uni_xxxx"
```

**Response:**

```json
{
  "issuer_id": "issuer:acmecorp.ai",
  "name": "Production Agents",
  "description": "Signs all production agent credentials",
  "trust_tier": "internal",
  "public_key": "base64_ed25519_public_key...",

  "status": "active",
  "created_at": "2026-01-15T00:00:00Z",
  "updated_at": "2026-01-15T00:00:00Z"

```

## Update Issuer

Update an issuer's name or description. The keypair and issuer ID cannot be changed.

```
PUT /api/issuers/:issuer_id
```

**Required scope:** `issuers:update`

**Request Body:**

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | New display name |
| `description` | string | Updated description |

**Example Request:**

```bash
curl -X PUT https://uniplex.ai/api/issuers/issuer:acmecorp.ai \
  -H "Authorization: Bearer uni_xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Production Agents v2",
    "description": "Updated for Q2 2026 deployment"
  }'
```

**Response:**

```json
{
  "issuer_id": "issuer:acmecorp.ai",
  "name": "Production Agents v2",
  "description": "Updated for Q2 2026 deployment",
  "status": "active",
  "updated_at": "2026-02-24T10:30:00Z"
}
```

## Revoke Issuer

Revoke an issuer. All passports signed by this issuer will immediately become invalid.

```
DELETE /api/issuers/:issuer_id
```

**Required scope:** `issuers:update` (revocation requires update scope, not delete)

<Warning>
  Issuer revocation is **immediate and irreversible**. All agents using passports from this issuer will lose access instantly. Ensure you have migrated all agents to a new issuer before revoking.
</Warning>

**Example Request:**

```bash
curl -X DELETE https://uniplex.ai/api/issuers/issuer:acmecorp.ai \
  -H "Authorization: Bearer uni_xxxx"
```

**Response:**

```json
{
  "issuer_id": "issuer:acmecorp.ai",
  "status": "revoked",
  "revoked_at": "2026-02-24T11:00:00Z",
  "passports_invalidated": 12
}
```

---

## Error Responses

| Error Code | Description |
|-----------|-------------|
| `ISSUER_NOT_FOUND` | No issuer with this ID exists |
| `ISSUER_ID_TAKEN` | The issuer_id is already in use |
| `ISSUER_ALREADY_REVOKED` | Cannot revoke an already-revoked issuer |
| `ISSUER_HAS_ACTIVE_PASSPORTS` | Warning: revocation will invalidate N passports |
