# Passports API

Passports are the cryptographic credentials that AI agents carry to prove their identity and permissions. This API covers issuing, reading, revoking passports, and the core authorization endpoint.

**Base URL:** `https://uniplex.ai/api`

---

## Issue Passport

Create a new passport for an AI agent. The Ed25519 keypair is generated server-side.

```
POST /api/passports
```

**Required scope:** `passports:issue`

**Request Body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `agent_id` | string | ✅ | Slug identifier (alphanumeric, hyphens) |
| `agent_name` | string | ✅ | Human-readable display name |
| `issuer_id` | string | ✅ | The issuer that signs this passport |
| `permissions` | string[] | ✅ | Granted permission namespaces |
| `expires_in` | string | ❌ | Duration like `"90d"`, `"1y"` (default: `"365d"`) |
| `public_key` | string | ❌ | Provide your own public key (agent generates its own keypair) |

**Example Request:**

```bash
curl -X POST https://uniplex.ai/api/passports \
  -H "Authorization: Bearer uni_xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "research-bot",
    "agent_name": "Research Bot",
    "issuer_id": "issuer:mycompany.ai",
    "permissions": ["web:search", "web:read", "mcp:read"],
    "expires_in": "90d"
  }'
```

**Response:**

```json
{
  "passport_id": "uni_abc123",
  "agent_id": "research-bot",
  "agent_name": "Research Bot",
  "issuer_id": "issuer:mycompany.ai",
  "permissions": ["web:search", "web:read", "mcp:read"],
  "public_key": "base64_ed25519_public_key...",
  "private_key": "base64_ed25519_private_key_SHOWN_ONCE",
  "issued_at": "2026-02-24T10:00:00Z",
  "expires_at": "2026-05-24T10:00:00Z",
  "status": "active",
  "passport_token": "base64_encoded_passport_json_for_gate_use..."
}
```

<Warning>
  `private_key` is shown exactly once. Store it in `UNIPLEX_PASSPORT_KEY` in your agent's environment immediately.
</Warning>

## List Passports

Retrieve passports for the authenticated account.

```
GET /api/passports
```

**Required scope:** `passports:read`

**Query Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `agent_id` | string | Filter by agent ID |
| `issuer_id` | string | Filter by issuer |
| `status` | string | `active`, `expired`, or `revoked` |
| `expiring_before` | string | ISO 8601 — passports expiring before this date |
| `limit` | integer | Results per page (default: 50, max: 200) |
| `offset` | integer | Pagination offset |

**Example Request:**

```bash
# List active passports expiring in the next 30 days
curl "https://uniplex.ai/api/passports" \
  -H "Authorization: Bearer uni_xxxx" \
  -G \
  --data-urlencode "status=active" \
  --data-urlencode "expiring_before=2026-03-24T00:00:00Z"
```

**Response:**

```json
{
  "passports": [
    {
      "passport_id": "uni_abc123",
      "agent_id": "research-bot",
      "agent_name": "Research Bot",
      "issuer_id": "issuer:mycompany.ai",
      "permissions": ["web:search", "web:read"],
      "issued_at": "2026-02-24T10:00:00Z",
      "expires_at": "2026-05-24T10:00:00Z",
      "status": "active"
    }
  ],
  "total": 1,
  "limit": 50,
  "offset": 0
}
```

## Get Passport

Retrieve a single passport by ID.

```
GET /api/passports/:passport_id
```

**Required scope:** `passports:read`

**Example Request:**

```bash
curl https://uniplex.ai/api/passports/uni_abc123 \
  -H "Authorization: Bearer uni_xxxx"
```

**Response:**

```json
{
  "passport_id": "uni_abc123",
  "agent_id": "research-bot",
  "agent_name": "Research Bot",
  "issuer_id": "issuer:mycompany.ai",
  "permissions": ["web:search", "web:read", "mcp:read"],
  "public_key": "base64_ed25519_public_key...",
  "issued_at": "2026-02-24T10:00:00Z",
  "expires_at": "2026-05-24T10:00:00Z",
  "status": "active",
  "gate_checks_count": 342,
  "last_used_at": "2026-02-24T15:30:00Z"
}
```

## Revoke Passport

Revoke a passport immediately. All future gate checks will deny this passport.

```
DELETE /api/passports/:passport_id
```

**Required scope:** `passports:revoke`

**Example Request:**

```bash
curl -X DELETE https://uniplex.ai/api/passports/uni_abc123 \
  -H "Authorization: Bearer uni_xxxx"
```

**Response:**

```json
{
  "passport_id": "uni_abc123",
  "status": "revoked",
  "revoked_at": "2026-02-24T16:00:00Z"
}
```

## Check Authorization

**This is the core endpoint.** Check whether a passport token is authorized to perform an action, evaluated against a gate's configuration and constraints.

```
POST /api/authorize
```

**Required scope:** Any valid API key (no specific scope required for this endpoint)

This is intentionally open — any authenticated caller can check authorization. The security comes from the passport and gate configuration, not the API key scope.

### Request

```json
{
  "passport_token": "base64_encoded_passport_json_token",
  "action": "mcp:search",
  "resource": "web:google",
  "gate_id": "gate_xxxxxxxx",
  "metadata": {
    "cost": "2.50",
    "user_id": "usr_123",
    "session_id": "sess_abc",
    "nonce": "unique-request-nonce-12345"
  }
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `passport_token` | string | ✅ | Base64-encoded passport token from the agent |
| `action` | string | ✅ | The action to check (e.g., `web:search`, `payments:send`) |
| `resource` | string | ❌ | Target resource for domain/URL constraint evaluation |
| `gate_id` | string | ❌ | Gate to evaluate against (uses account default if omitted) |
| `metadata` | object | ❌ | Additional context: cost, user_id, nonce, etc. |

### Response (Allow)

```json
{
  "allowed": true,
  "reason": "All constraints satisfied",
  "attestation_id": "att_xxxxxxxx",
  "checked_at": "2026-02-24T10:30:00Z",
  "constraints_evaluated": [
    {
      "type": "max_per_day",
      "value": "$50.00",
      "current": "$12.40",
      "result": "pass",
      "detail": "Daily spend $12.40 of $50.00 limit"
    },
    {
      "type": "domain_allowlist",
      "value": "google.com,bing.com",
      "current": "google.com",
      "result": "pass",
      "detail": "Target domain google.com is in allowlist"
    }
  ]
}
```

### Response (Deny)

```json
{
  "allowed": false,
  "reason": "Daily spending limit exceeded: $48.75 of $50.00 used",
  "attestation_id": "att_yyyyyyyy",
  "checked_at": "2026-02-24T22:45:00Z",
  "constraints_evaluated": [
    {
      "type": "max_per_day",
      "value": "$50.00",
      "current": "$48.75",
      "result": "fail",
      "detail": "Would exceed daily limit: $48.75 + $5.00 = $53.75 > $50.00"
    }
  ]
}
```

### Example Request

```bash
curl -X POST https://uniplex.ai/api/authorize \
  -H "Authorization: Bearer uni_xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "passport_token": "eyJwYXlsb2FkIjp7InBhc3Nwb3J0X2lkIjoiI...",
    "action": "mcp:search",
    "resource": "web:google.com",
    "gate_id": "gate_def456",
    "metadata": {
      "cost": "0.10",
      "session_id": "sess_abc123"
    }
  }'
```

## Error Responses

| Error Code | HTTP Status | Description |
|-----------|------------|-------------|
| `PASSPORT_NOT_FOUND` | 404 | Passport ID does not exist |
| `PASSPORT_EXPIRED` | 400 | Passport has passed its `expires_at` |
| `PASSPORT_REVOKED` | 400 | Passport was explicitly revoked |
| `PASSPORT_SIGNATURE_INVALID` | 400 | The passport's Ed25519 signature is invalid |
| `ISSUER_NOT_TRUSTED` | 403 | Passport issuer not in gate's trusted issuers |
| `PERMISSION_DENIED` | 403 | Passport lacks the required permission |
| `CONSTRAINT_VIOLATION` | 403 | A guardrail constraint blocked the action |
| `GATE_NOT_FOUND` | 404 | The specified gate_id does not exist |
