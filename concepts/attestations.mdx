# Attestations

An **attestation** is an immutable, cryptographically signed record of a gate decision. Every time a gate allows or denies an action, it produces an attestation — a document that proves exactly what happened, when, and why, and that this record has not been altered since it was created.

Attestations are the audit trail. They're what you show to a compliance auditor, a client asking "did the agent access my data?", or a developer debugging why an agent was blocked.

---

## Why Immutability Matters

Attestations are signed by the gate's private key at the moment of the decision. The signature covers the entire document content. If any field — the decision, the timestamp, the reason, the agent ID — is modified after the fact, the signature verification fails.

This means:
- You can share an attestation with anyone and they can independently verify it hasn't been tampered with
- Supravisor cannot retroactively alter what an agent was or wasn't allowed to do
- Your agent cannot dispute its own action history
- Third parties can verify the record without trusting Supravisor's word

## Attestation Anatomy

```json
{
  "attestation_id": "att_xyz789",
  "decision": "allow",
  "reason": "All constraints satisfied",
  "passport_id": "uni_abc123",
  "agent_id": "my-research-agent",
  "agent_name": "Research Bot",
  "gate_id": "gate_def456",
  "gate_name": "Research Gate",
  "action": "mcp:search",
  "resource": "web:google",
  "timestamp": "2026-02-24T10:30:00Z",
  "constraints_evaluated": [
    {
      "type": "max_per_day",
      "value": "$50.00",
      "current": "$12.40",
      "result": "pass",
      "detail": "Daily spend $12.40 of $50.00 limit"
    },
    {
      "type": "max_per_minute",
      "value": "10",
      "current": "3",
      "result": "pass",
      "detail": "3 actions in current minute window"
    },
    {
      "type": "domain_allowlist",
      "value": "openai.com,anthropic.com,google.com",
      "current": "google.com",
      "result": "pass",
      "detail": "Target domain google.com is in allowlist"
    }
  ],
  "issuer_id": "issuer:mycompany.ai",
  "trust_profile": "L2",
  "nonce": "req_nonce_abc",
  "gate_signature": "Ed25519-base64_encoded_signature_over_all_fields..."
}
```

| Field | Description |
|-------|-------------|
| `attestation_id` | Globally unique ID, prefixed with `att_` |
| `decision` | `"allow"` or `"deny"` |
| `reason` | Human-readable explanation — always present |
| `passport_id` | The passport that was presented |
| `agent_id` | The agent slug for easy filtering |
| `gate_id` | Which gate made this decision |
| `action` | The action that was requested |
| `resource` | The resource the action targeted (if provided) |
| `timestamp` | ISO 8601 UTC — when the decision was made |
| `constraints_evaluated` | Array of every constraint that was checked |
| `trust_profile` | L1, L2, or L3 |
| `gate_signature` | Ed25519 signature over the entire document |

### Denial Attestation Example

```json
{
  "attestation_id": "att_abc456",
  "decision": "deny",
  "reason": "Spending limit exceeded: $48.75 of $50.00 daily limit used",
  "passport_id": "uni_abc123",
  "agent_id": "my-research-agent",
  "gate_id": "gate_def456",
  "action": "payments:send",
  "resource": "stripe:charge_5.00",
  "timestamp": "2026-02-24T22:45:00Z",
  "constraints_evaluated": [
    {
      "type": "max_per_day",
      "value": "$50.00",
      "current": "$48.75",
      "result": "fail",
      "detail": "Would exceed daily limit: $48.75 + $5.00 = $53.75 > $50.00"
    }
  ],
  "gate_signature": "Ed25519-base64..."
}
```

## Accessing via API

```bash
# List recent attestations
curl https://uniplex.ai/api/audit-trail \
  -H "Authorization: Bearer uni_xxxx" \
  -G \
  --data-urlencode "agent_id=my-research-agent" \
  --data-urlencode "limit=50"

# Get a specific attestation
curl https://uniplex.ai/api/audit-trail/att_xyz789 \
  -H "Authorization: Bearer uni_xxxx"

# Filter by decision
curl https://uniplex.ai/api/audit-trail \
  -H "Authorization: Bearer uni_xxxx" \
  -G \
  --data-urlencode "decision=deny" \
  --data-urlencode "from=2026-02-01T00:00:00Z" \
  --data-urlencode "to=2026-02-28T23:59:59Z"
```

## Sharing an Attestation

Every attestation can be shared via a public URL:

```bash
curl -X POST https://uniplex.ai/api/audit-trail/att_xyz789/share \
  -H "Authorization: Bearer uni_xxxx"
```

```json
{
  "share_url": "https://uniplex.ai/proof/att_xyz789_sharetoken",
  "expires_at": "2026-03-24T10:30:00Z"
}
```

**What the public URL shows:**
- Decision (allow/deny)
- Action and resource
- Timestamp
- Constraints evaluated and their results
- Gate signature for independent verification
- Agent name (not the passport private key or API key)

**What the public URL hides:**
- Your API keys
- Internal gate configuration details
- Issuer private keys
- Any personally identifiable information beyond what's in the attestation

<Tip>
  Share attestation URLs with clients as proof that their agent operated within its authorized scope. The cryptographic signature lets them verify the record independently.
</Tip>

## Use Cases

### Compliance Audits

Regulators and auditors often need evidence that automated systems operated within approved boundaries. Export your attestation history for any time period as a signed JSON file and hand it over. Each record's signature proves it's unaltered.

```bash
# Export a month's attestations as CSV for an auditor
curl "https://uniplex.ai/api/audit-trail/export?format=csv&from=2026-01-01&to=2026-01-31" \
  -H "Authorization: Bearer uni_xxxx" \
  -o january-audit.csv
```

### Client Reporting

Running agents on behalf of clients? Share a public URL for any attestation to prove the agent only did what was authorized, spent within budget, and never accessed restricted data.

### Debugging Agent Behavior

An agent got blocked and you don't know why? Find the denial attestation — it tells you exactly which constraint triggered, with current values, so you can fix the configuration or explain the behavior.

### Dispute Resolution

If a client disputes an action ("the agent made a charge I didn't authorize"), pull the attestation for that action. It shows the decision (allow/deny), the constraint evaluations (was the spending limit respected?), and the exact timestamp — all with a signature you can verify.

### Billing Verification

For agents that make paid API calls or spend on behalf of users, attestations provide an independent record of every spending decision. Cross-reference attestations against invoices to verify billing accuracy.

## Verifying a Signature Independently

You can verify any attestation's signature without trusting Supravisor's API:

```python
import json
import base64
from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PublicKey

def verify_attestation(attestation: dict, gate_public_key_b64: str) -> bool:
    signature_b64 = attestation.pop("gate_signature")
    signature = base64.b64decode(signature_b64)
    
    # Canonical JSON (sorted keys, no whitespace)
    payload = json.dumps(attestation, sort_keys=True, separators=(',', ':')).encode()
    
    public_key_bytes = base64.b64decode(gate_public_key_b64)
    public_key = Ed25519PublicKey.from_public_bytes(public_key_bytes)
    
    try:
        public_key.verify(signature, payload)
        return True
    except Exception:
        return False
```
