# Issuers

An **issuer** is the root of trust in Supravisor's credential chain. Every passport must be signed by an issuer, and gates verify passports by checking whether the signing issuer is in their trusted list. Think of an issuer as your own certificate authority — but purpose-built for AI agent identity rather than web TLS.

When a gate receives a passport, it answers one question: *"Do I trust the entity that signed this?"* If yes, it proceeds to evaluate permissions and guardrails. If not, the request is denied before any rules are even checked.

---

## Issuer ID Format

Every issuer has a globally unique ID in the format:

```
issuer:yourdomain.ai
```

For personal use or testing:
```
issuer:personal.local
issuer:dev.myname
```

For organizations:
```
issuer:acmecorp.ai
issuer:engineering.acmecorp.ai
```

The format is intentionally human-readable. When you look at an attestation, you can immediately see which authority signed the credential.

## How Issuers Relate to Passports

The relationship is one-to-many: **one issuer signs many passports**.

```
issuer:acmecorp.ai
  ├── passport: research-agent-001
  ├── passport: email-agent-prod
  ├── passport: customer-support-bot
  └── passport: data-pipeline-worker
```

When you revoke an issuer, all passports it signed become untrusted. When you issue a new passport, you choose which issuer signs it. Gates are configured to trust specific issuers — you might have a production gate that only trusts `issuer:acmecorp.ai` and a dev gate that also accepts `issuer:dev.acmecorp.ai`.

## Client-Side Key Generation

<Note>
  Supravisor never sees your issuer's private key. The Ed25519 keypair is generated entirely in your browser using the Web Crypto API. The private key is shown to you once and then discarded from Supravisor's systems.
</Note>

When you create an issuer:

1. Your browser generates an Ed25519 keypair locally
2. The **public key** is sent to Supravisor and stored
3. The **private key** is displayed in the UI for you to copy
4. Supravisor stores only the public key — it cannot sign passports on your behalf

This means passport forgery is cryptographically impossible even if Supravisor's servers were compromised. Only the holder of the private key can create valid passports.

## Creating an Issuer

<Steps>
  <Step title="Open the Issuers page">
    In the Supravisor dashboard, click **Issuers** in the left sidebar.
  </Step>
  <Step title="Click New Issuer">
    Click the **+ New Issuer** button in the top right.
  </Step>
  <Step title="Fill in issuer details">
    - **Name:** Human-readable label (e.g., "Production Agents")
    - **Issuer ID:** The `issuer:yourdomain.ai` format identifier
    - **Description:** Optional notes about what this issuer is for
  </Step>
  <Step title="Generate the keypair">
    Click **Generate Keys**. Your browser will create an Ed25519 keypair. The public key is automatically uploaded. The private key appears in a highlighted box.
  </Step>
  <Step title="Save the private key">
    Copy the private key immediately and store it in a secrets manager (1Password, Vault, AWS Secrets Manager, etc.). Click **I've saved my key** to confirm.

    <Warning>
      This is the only time the private key is shown. If you close this window without saving it, you cannot recover it — you'll need to create a new issuer.
    </Warning>
  </Step>
</Steps>

## Managing Issuers

### Listing Issuers

```bash
curl https://uniplex.ai/api/issuers \
  -H "Authorization: Bearer uni_xxxx"
```

```json
{
  "issuers": [
    {
      "issuer_id": "issuer:acmecorp.ai",
      "name": "Production Agents",
      "trust_tier": "internal",
      "passport_count": 12,
      "created_at": "2026-01-15T00:00:00Z",
      "status": "active"
    }
  ]
}
```

### Revoking an Issuer

Revoking an issuer immediately invalidates **all passports** signed by it. Gates will deny any passport from a revoked issuer, even if the passport itself hasn't expired.

```bash
curl -X DELETE https://uniplex.ai/api/issuers/issuer:acmecorp.ai \
  -H "Authorization: Bearer uni_xxxx"
```

<Warning>
  Issuer revocation is immediate and irreversible. All agents using passports from this issuer will lose access instantly. Have a migration plan before revoking production issuers.
</Warning>

## When to Create Multiple Issuers

Using separate issuers gives you isolation, auditability, and the ability to revoke a group of agents without affecting others.

| Scenario | Recommended Structure |
|----------|----------------------|
| Dev vs. production | `issuer:dev.company.ai` + `issuer:company.ai` |
| Multiple teams | `issuer:eng.company.ai` + `issuer:ops.company.ai` |
| Security incident | Revoke the compromised issuer, reissue from new one |
| Client isolation | `issuer:client-a.company.ai` per client |
| Vendor/partner agents | Separate issuer with restricted trust scope |

## Verifying Issuer Trust Level

<CodeGroup>
```python Python
from uniplex import UniplexClient

client = UniplexClient(api_key="uni_xxxx")
issuer = client.issuers.get("issuer:acmecorp.ai")

print(f"Trust tier: {issuer.trust_tier}")   # "internal"
print(f"Status: {issuer.status}")           # "active"
print(f"Passports issued: {issuer.passport_count}")  # 12

# Verify a passport comes from a trusted issuer
from uniplex import Gate, TrustProfile

gate = Gate(
    profile=TrustProfile.L2,
    trusted_issuers=["issuer:acmecorp.ai"]
)

result = gate.verify(passport_token)
print(f"Issuer trusted: {result.issuer_trusted}")  # True
print(f"Issuer ID: {result.issuer_id}")            # "issuer:acmecorp.ai"
```

```typescript TypeScript
import { UniplexClient, Gate, TrustProfile } from 'uniplex';

const client = new UniplexClient({ apiKey: 'uni_xxxx' });
const issuer = await client.issuers.get('issuer:acmecorp.ai');

console.log(`Trust tier: ${issuer.trustTier}`);     // "internal"
console.log(`Status: ${issuer.status}`);             // "active"
console.log(`Passports issued: ${issuer.passportCount}`); // 12

// Verify a passport comes from a trusted issuer
const gate = new Gate({
  profile: TrustProfile.L2,
  trustedIssuers: ['issuer:acmecorp.ai'],
});

const result = await gate.verify(passportToken);
console.log(`Issuer trusted: ${result.issuerTrusted}`); // true
console.log(`Issuer ID: ${result.issuerId}`);           // "issuer:acmecorp.ai"
```
