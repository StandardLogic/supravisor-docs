# Trust Profiles

Every gate operates at one of three **trust profiles**: L1 (Baseline), L2 (Standard), or L3 (Strict). The profile determines how rigorously the gate verifies a passport — what it checks, what it skips, and how much it trusts the token it receives.

Choose the profile that matches your risk tolerance. There's no penalty for starting at L1 during development and upgrading to L2 or L3 for production.

---

## Quick Comparison

| Feature | L1 Baseline | L2 Standard | L3 Strict |
|---------|------------|-------------|-----------|
| Self-issued passports | ✅ Accepted | ❌ Requires managed issuer | ❌ Requires managed issuer |
| Signature verification | ✅ | ✅ | ✅ |
| Issuer trust check | ❌ Any issuer | ✅ Trusted list only | ✅ Trusted list only |
| Replay attack protection | ❌ | ✅ Nonce-based | ✅ Proof-of-possession |
| Revocation check | ❌ | ✅ Cached (60s) | ✅ Live (every request) |
| Network call required | ❌ Never | ❌ Usually not | ✅ Per-request (revocation) |
| Typical latency | <0.1ms | <0.5ms | <2ms |

## L1 — Baseline

L1 is the "just make it work" profile. It verifies that a passport is structurally valid and correctly self-signed, but doesn't check who signed it or whether it's been revoked.

**What L1 accepts:**
- Self-issued passports (created with `Agent.create()` — no Supravisor account needed)
- Passports from any issuer, trusted or not
- Any non-expired, non-malformed passport

**What L1 checks:**
- ✅ Ed25519 signature is valid for the payload
- ✅ Passport is not expired
- ✅ Requested action is in the permission list
- ✅ Guardrail constraints (spending, rate, domain, etc.)

**What L1 skips:**
- ❌ Issuer trust verification
- ❌ Replay attack protection
- ❌ Revocation list check

**Best for:** Local development, unit tests, internal scripts, quick experiments, prototyping.

<Warning>
  Never use L1 in production. Anyone who creates a self-issued passport can access your L1 gate. It offers guardrail enforcement but no identity assurance.
</Warning>

### L1 Code Examples

<CodeGroup>
```python Python
from uniplex import Agent, Gate, TrustProfile

# Create a self-issued agent (no account needed)
agent = Agent.create("dev-agent", permissions=["web:search", "mcp:read"])

# Create an L1 gate (no trusted_issuers needed)
gate = Gate(profile=TrustProfile.L1)

# Authorize — will pass for any valid self-issued passport
token = agent.export_token()
decision = gate.authorize(token, "web:search")
print(decision.allowed)  # True
```

```typescript TypeScript
import { Agent, Gate, TrustProfile } from 'uniplex';

// Create a self-issued agent (no account needed)
const agent = await Agent.create('dev-agent', {
  permissions: ['web:search', 'mcp:read'],
});

// Create an L1 gate
const gate = new Gate({ profile: TrustProfile.L1 });

// Authorize
const token = agent.exportToken();
const decision = await gate.authorize(token, 'web:search');
console.log(decision.allowed); // true
```
</CodeGroup>

## L2 — Standard

L2 is the recommended profile for all production deployments. It adds issuer trust verification and nonce-based replay protection — meaning a captured token can't be replayed to gain unauthorized access.

**What L2 accepts:**
- Only passports signed by issuers in the gate's `trusted_issuers` list
- Tokens with a valid, unused nonce

**What L2 checks:**
- ✅ Everything L1 checks
- ✅ Issuer is in the trusted list
- ✅ Nonce hasn't been seen before (replay protection)
- ✅ Revocation status (checked against Supravisor's cached revocation list)

**What L2 skips:**
- ❌ Proof-of-possession (the agent doesn't need to sign the request)

**Best for:** Production agents, team deployments, any service that faces external networks, agents handling sensitive data or performing actions with side effects.

### L2 Code Examples

<CodeGroup>
```python Python
from uniplex import Agent, Gate, TrustProfile

# Load an agent with a managed passport
agent = Agent.from_env()  # reads UNIPLEX_PASSPORT_KEY

# Create an L2 gate with trusted issuers
gate = Gate(
    profile=TrustProfile.L2,
    trusted_issuers=["issuer:mycompany.ai"],
    constraints={
        "max_per_day": "$50.00",
        "domain_blocklist": "facebook.com,twitter.com"
    }
)

# Authorize — verifies issuer + nonce + constraints
token = agent.generate_token()  # includes a nonce
decision = gate.authorize(token, "web:search")

if decision.allowed:
    print(f"Authorized. Attestation: {decision.attestation_id}")
else:
    print(f"Denied: {decision.reason}")
```

```typescript TypeScript
import { Agent, Gate, TrustProfile } from 'uniplex';

// Load an agent with a managed passport
const agent = await Agent.fromEnv();

// Create an L2 gate
const gate = new Gate({
  profile: TrustProfile.L2,
  trustedIssuers: ['issuer:mycompany.ai'],
  constraints: {
    maxPerDay: '$50.00',
    domainBlocklist: 'facebook.com,twitter.com',
  },
});

// Authorize
const token = agent.generateToken(); // includes a nonce
const decision = await gate.authorize(token, 'web:search');

if (decision.allowed) {
  console.log(`Authorized. Attestation: ${decision.attestationId}`);
} else {
  console.log(`Denied: ${decision.reason}`);
}
```
</CodeGroup>

## L3 — Strict

L3 is the highest security level. It requires **proof-of-possession**: the agent must cryptographically sign each request with its private key, proving it physically holds the key — not just a copy of the token. Combined with live revocation checks, L3 makes token theft essentially useless.

**What L3 accepts:**
- Passports signed by trusted issuers
- Requests where the agent has signed the request payload with its private key
- Only non-revoked passports (checked live against Supravisor's revocation list)

**What L3 checks:**
- ✅ Everything L2 checks
- ✅ Proof-of-possession: agent signs `{passport_id}:{nonce}:{timestamp}` with its private key
- ✅ Live revocation check (not cached — every single request)
- ✅ Timestamp is within 30 seconds to prevent delayed replay

**Best for:** Financial agents, healthcare automation, payroll, crypto trading, anything where the consequences of unauthorized access are severe.

<Note>
  L3 requires a network call for the live revocation check. This typically adds 10–50ms depending on latency to Supravisor's revocation service.
</Note>

### L3 Code Examples

<CodeGroup>
```python Python
from uniplex import Agent, Gate, TrustProfile

# L3 requires the full keypair (not just passport_id)
agent = Agent.from_env()  # reads UNIPLEX_PASSPORT_KEY

# Create an L3 gate
gate = Gate(
    profile=TrustProfile.L3,
    trusted_issuers=["issuer:fintech.mycompany.ai"],
    constraints={
        "max_per_action": "$100.00",
        "max_per_day": "$500.00",
        "approval_threshold": "$50.00",
        "no_pii_export": True,
        "action_blocklist": "payments:refund,payments:void"
    }
)

# Agent signs the request — proof-of-possession
token = agent.generate_signed_token(nonce="unique-request-nonce")
decision = gate.authorize(token, "payments:send", resource="stripe:charge")

if not decision.allowed:
    # Log the denial with attestation for audit
    logger.warning(f"Payment denied: {decision.reason} (att: {decision.attestation_id})")
```

```typescript TypeScript
import { Agent, Gate, TrustProfile } from 'uniplex';

const agent = await Agent.fromEnv();

const gate = new Gate({
  profile: TrustProfile.L3,
  trustedIssuers: ['issuer:fintech.mycompany.ai'],
  constraints: {
    maxPerAction: '$100.00',
    maxPerDay: '$500.00',
    approvalThreshold: '$50.00',
    noPiiExport: true,
    actionBlocklist: 'payments:refund,payments:void',
  },
});

// Agent signs the request — proof-of-possession
const token = agent.generateSignedToken({ nonce: 'unique-request-nonce' });
const decision = await gate.authorize(token, 'payments:send', {
  resource: 'stripe:charge',
});

if (!decision.allowed) {
  console.error(`Payment denied: ${decision.reason} (att: ${decision.attestationId})`);
}
```
</CodeGroup>

## Choosing the Right Profile

```
Is this for development or testing only?
  ├── Yes → L1
  └── No ↓

Does this agent handle money, health data, or high-stakes decisions?
  ├── Yes → L3
  └── No ↓

Is this agent externally facing or part of a team deployment?
  ├── Yes → L2
  └── Internal-only → L2 (still recommended)
```

When in doubt, use L2. It adds minimal overhead and provides meaningful protection against the most common attack vectors (token theft, replay attacks, rogue agents).
