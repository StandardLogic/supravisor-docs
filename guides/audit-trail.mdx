# Audit Trail and Session Sharing

Every gate decision — allow or deny — produces a signed attestation record stored in the audit trail. This is not a log file that can be edited. It's a cryptographic record: each entry is signed by the gate's private key, and the signature covers all fields. Tampering with any field breaks the signature.

The audit trail is what turns "trust me, the agent behaved" into "here's proof the agent behaved."

---

## What's in the Audit Trail

Every entry contains:

| Field | Description |
|-------|-------------|
| Attestation ID | Unique ID (`att_`) for this decision |
| Decision | `allow` or `deny` |
| Reason | Exact explanation (which constraint passed/failed) |
| Agent ID + Name | Which agent made the request |
| Gate ID + Name | Which gate evaluated it |
| Action | What the agent tried to do |
| Resource | What target resource (if provided) |
| Timestamp | UTC timestamp, millisecond precision |
| Constraints evaluated | Array of every constraint checked, with current values |
| Gate signature | Ed25519 signature over the entire record |

---

## Reading It in the Dashboard

### Activity Tab

The Activity tab shows a live stream of recent decisions, most recent first:

- 🟢 Green rows = allowed actions
- 🔴 Red rows = denied actions
- Each row shows: agent name, action, decision, timestamp
- Click any row to see the full attestation with constraints breakdown

**Best for:** Real-time monitoring, quick spot checks, seeing what just happened.

### Audit Trail Tab

The Audit Trail tab shows the complete queryable history:

- Filter by agent, gate, decision, time range
- Sort by any column
- Paginated (50 per page by default)
- Export button (JSON or CSV)
- Share button on individual rows

**Best for:** Compliance exports, debugging specific incidents, client reporting.

---

## Filtering

In the dashboard or via API, filter by:

- `agent_id` — a specific agent
- `gate_id` — decisions from a specific gate
- `decision` — `allow` or `deny` only
- `from` / `to` — time range (ISO 8601)
- `action` — a specific action like `web:search`
- `limit` — number of results (max 500 per request)

## API Access

```bash
# Basic: last 50 decisions
curl https://uniplex.ai/api/audit-trail \
  -H "Authorization: Bearer uni_xxxx" \
  -G --data-urlencode "limit=50"

# Filtered: all denials for a specific agent
curl https://uniplex.ai/api/audit-trail \
  -H "Authorization: Bearer uni_xxxx" \
  -G \
  --data-urlencode "agent_id=my-research-agent" \
  --data-urlencode "decision=deny" \
  --data-urlencode "limit=50"

# Time-ranged: all decisions in February 2026
curl https://uniplex.ai/api/audit-trail \
  -H "Authorization: Bearer uni_xxxx" \
  -G \
  --data-urlencode "from=2026-02-01T00:00:00Z" \
  --data-urlencode "to=2026-02-28T23:59:59Z" \
  --data-urlencode "limit=500"

# Specific action type
curl https://uniplex.ai/api/audit-trail \
  -H "Authorization: Bearer uni_xxxx" \
  -G \
  --data-urlencode "action=payments:send"

# Get a single attestation
curl https://uniplex.ai/api/audit-trail/att_xyz789 \
  -H "Authorization: Bearer uni_xxxx"
```

### Python SDK

```python
from uniplex import UniplexClient
from datetime import datetime, timedelta

client = UniplexClient(api_key="uni_xxxx")

# Last 50 decisions for an agent
attestations = client.audit_trail.list(
    agent_id="my-research-agent",
    limit=50
)

# Denials this week
week_ago = datetime.utcnow() - timedelta(days=7)
denials = client.audit_trail.list(
    decision="deny",
    from_time=week_ago,
    limit=200
)

for att in denials:
    print(f"{att.timestamp} | {att.agent_id} | {att.action} | {att.reason}")

# Get a specific attestation
att = client.audit_trail.get("att_xyz789")
print(att.gate_signature)  # Ed25519 signature for independent verification
```

## Exporting as JSON or CSV

### Dashboard

1. Navigate to **Audit Trail**
2. Apply your filters
3. Click **Export** → choose JSON or CSV
4. File downloads immediately

### API

```bash
# Export as CSV
curl "https://uniplex.ai/api/audit-trail/export?format=csv&agent_id=my-agent&from=2026-01-01" \
  -H "Authorization: Bearer uni_xxxx" \
  -o agent-audit-jan-2026.csv

# Export as JSON
curl "https://uniplex.ai/api/audit-trail/export?format=json&from=2026-01-01&to=2026-01-31" \
  -H "Authorization: Bearer uni_xxxx" \
  -o january-audit.json
```

The JSON export includes the full attestation for each record, including the `gate_signature`. The CSV export includes all fields as columns.

## Sharing: Public Proof URLs

You can generate a public URL for any attestation. Anyone with the URL can view the record and independently verify the cryptographic signature.

### Creating a Share Link

```bash
curl -X POST https://uniplex.ai/api/audit-trail/att_xyz789/share \
  -H "Authorization: Bearer uni_xxxx" \
  -H "Content-Type: application/json" \
  -d '{"expires_in": "30d"}'
```

```json
{
  "share_url": "https://uniplex.ai/proof/att_xyz789_abc123token",
  "expires_at": "2026-03-24T10:30:00Z"
}
```

### What the Public URL Shows

- ✅ Decision (allow/deny)
- ✅ Action and resource
- ✅ Timestamp
- ✅ Constraints evaluated (with pass/fail for each)
- ✅ Agent name
- ✅ Gate name
- ✅ Gate signature for independent verification

### What the Public URL Hides

- ❌ Your API keys
- ❌ Gate configuration details
- ❌ Issuer private keys
- ❌ Sensitive metadata fields

## Use Cases

### Compliance Audits

Regulators and compliance teams need evidence that AI systems operated within approved boundaries.

**Workflow:**
1. Export audit trail for the compliance period as JSON
2. Provide the export to auditors
3. Each record's `gate_signature` lets auditors independently verify tamper-proof records
4. Include gate public keys (from dashboard → Gates → Export Public Key) so auditors can verify signatures offline

```bash
# Export Q4 2025 for compliance audit
curl "https://uniplex.ai/api/audit-trail/export?format=json&from=2025-10-01&to=2025-12-31" \
  -H "Authorization: Bearer uni_xxxx" \
  -o q4-2025-compliance-export.json
```

---

### Client Reporting

Running agents on behalf of clients? Share proof that the agent only did what was authorized.

**Workflow:**
1. At the end of each client session, generate share links for any notable decisions
2. Include the links in your client report
3. The client can click each link to see the signed decision record

```python
# Generate share links for all decisions in a client session
client_attestations = client.audit_trail.list(
    agent_id="client-abc-research-agent",
    from_time=session_start
)

share_links = []
for att in client_attestations:
    share = client.audit_trail.share(att.attestation_id, expires_in="90d")
    share_links.append({
        "action": att.action,
        "decision": att.decision,
        "timestamp": att.timestamp,
        "proof_url": share.share_url
    })

# Include share_links in client report
```

### Debugging Agent Behavior

An agent got blocked or behaved unexpectedly? The audit trail tells you exactly what happened.

**Workflow:**
1. Find the denial attestation in the Activity tab
2. Expand the `constraints_evaluated` array
3. Find the constraint with `"result": "fail"`
4. Read the `detail` field for the exact current value and limit

```python
# Find the most recent denial for an agent
denial = client.audit_trail.list(
    agent_id="my-agent",
    decision="deny",
    limit=1
)[0]

print(f"Denied at: {denial.timestamp}")
print(f"Action: {denial.action}")
print(f"Reason: {denial.reason}")

for constraint in denial.constraints_evaluated:
    if constraint["result"] == "fail":
        print(f"\nFailed: {constraint['type']}")
        print(f"  Limit: {constraint['value']}")
        print(f"  Current: {constraint['current']}")
        print(f"  Detail: {constraint['detail']}")
```

### Dispute Resolution

If someone disputes an agent action ("the agent made a charge I didn't authorize" or "the agent accessed my private files"), pull the attestation.

The attestation proves:
- Whether the action was allowed or denied
- Which constraints were checked and what the values were
- The exact timestamp
- That this record hasn't been altered (via signature)

For disputed allow decisions, the attestation shows which constraint permitted the action. For disputed denials, it shows which constraint blocked it.

## Verifying a Signature Offline

Any attestation can be verified without trusting Supravisor's API. Get your gate's public key from the dashboard (Gates → your gate → Export Public Key):

```python
import json
import base64
from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PublicKey

def verify_attestation(attestation_json: str, gate_public_key_b64: str) -> bool:
    attestation = json.loads(attestation_json)
    signature_b64 = attestation.pop("gate_signature")
    signature = base64.b64decode(signature_b64)
    
    # Canonical JSON (sorted keys, no whitespace)
    payload = json.dumps(
        attestation, sort_keys=True, separators=(',', ':')
    ).encode('utf-8')
    
    public_key_bytes = base64.b64decode(gate_public_key_b64)
    public_key = Ed25519PublicKey.from_public_bytes(public_key_bytes)
    
    try:
        public_key.verify(signature, payload)
        return True
    except Exception:
        return False

# Usage:
with open("attestation.json") as f:
    attestation_data = f.read()

is_valid = verify_attestation(attestation_data, "YOUR_GATE_PUBLIC_KEY_BASE64")
print(f"Attestation valid: {is_valid}")
```
