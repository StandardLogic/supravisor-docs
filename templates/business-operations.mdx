# Business Operations Templates

Six templates for business process automation. Built for team deployments with escalation paths, audit requirements, and conservative defaults around sensitive operations.

---

## 1. Customer Support Agent

**Use case:** An AI that reads your knowledge base, responds to support tickets, and escalates to a human when specific keywords are detected (billing disputes, legal threats, refund requests).

| Constraint | Value | Why |
|-----------|-------|-----|
| `action_allowlist` | `tickets:read, tickets:respond, kb:read, tickets:tag` | No create/delete/close |
| `action_blocklist` | `tickets:delete, tickets:close, payments:*` | No payment access |
| `required` | `ticket_id` | Every action tagged to a ticket |
| `no_pii_export` | `true` | Customer data protection |

```json
{
  "template": "customer-support-agent",
  "constraints": {
    "action_allowlist": [
      "tickets:read",
      "tickets:respond",
      "tickets:tag",
      "tickets:escalate",
      "kb:read",
      "kb:search"
    ],
    "action_blocklist": [
      "tickets:delete",
      "tickets:close",
      "tickets:bulk_update",
      "payments:read",
      "payments:send"
    ],
    "required": "ticket_id",
    "no_pii_export": true,
    "max_per_minute": 10,
    "read_only": false
  }
}
```

<Tip>
  Add keyword-based escalation by configuring your ticket system to pause agent responses when terms like "refund," "lawyer," or "cancel account" are detected, before Supravisor evaluation.
</Tip>

## 2. Sales Development Agent

**Use case:** An AI that researches prospects and prepares outreach materials — but cannot send emails without a human approving each one. Research is fully automated; sending requires human review.

| Constraint | Value | Why |
|-----------|-------|-----|
| `action_allowlist` | `web:search, crm:read, crm:update, email:draft` | Can draft but not send |
| `action_blocklist` | `email:send, email:schedule` | No autonomous sending |
| `approval_threshold` | `$0.01` | Human approves everything above zero cost (i.e., all sends) |
| `no_pii_export` | `true` | No bulk prospect export |

```json
{
  "template": "sales-development-agent",
  "constraints": {
    "action_allowlist": [
      "web:search",
      "web:read",
      "crm:read",
      "crm:create",
      "crm:update",
      "email:draft",
      "email:read"
    ],
    "action_blocklist": [
      "email:send",
      "email:schedule",
      "email:bulk_send",
      "crm:delete"
    ],
    "no_pii_export": true,
    "max_per_hour": 50,
    "domain_allowlist": "linkedin.com,crunchbase.com,apollo.io",
    "required": "prospect_id"
  }
}
```

## 3. Contract Review Agent

**Use case:** An AI that reads contracts and legal documents, flags clause types (liability caps, auto-renewal, IP assignment), and produces summaries — without modifying any documents.

| Constraint | Value | Why |
|-----------|-------|-----|
| `read_only` | `true` | Absolute — no modifications |
| `action_allowlist` | `files:read, docs:read, docs:annotate_local` | Read only + local notes |
| `no_pii_export` | `true` | No customer PII in outputs |
| `domain_blocklist` | External services | No document upload to external AI |

```json
{
  "template": "contract-review-agent",
  "constraints": {
    "read_only": true,
    "action_allowlist": [
      "files:read",
      "docs:read",
      "docs:search",
      "docs:summarize"
    ],
    "action_blocklist": [
      "docs:write",
      "docs:sign",
      "docs:share_external",
      "files:upload_external"
    ],
    "domain_blocklist": "openai.com,anthropic.com,chatgpt.com",
    "no_pii_export": true,
    "required": "document_id"
  }
}
```

<Warning>
  The `domain_blocklist` for external AI services ensures that contract text is not sent to third-party AI services. If you want to use an AI service for summarization, add your specific API domain to the allowlist and audit the integration carefully.
</Warning>

## 4. HR Onboarding Agent

**Use case:** An AI that sends onboarding emails from templates, sets up accounts, and guides new hires — without access to salary data, SSNs, or the ability to delete HR records.

| Constraint | Value | Why |
|-----------|-------|-----|
| `action_allowlist` | `email:send_template, hris:read_public, accounts:create` | Template emails only |
| `action_blocklist` | `hris:read_salary, hris:read_ssn, hris:delete` | Hard blocks on sensitive fields |
| `no_pii_export` | `true` | No bulk export of employee data |
| `required` | `employee_id` | Every action tagged to an employee |

```json
{
  "template": "hr-onboarding-agent",
  "constraints": {
    "action_allowlist": [
      "email:send_template",
      "hris:read_public",
      "hris:update_onboarding_status",
      "accounts:provision",
      "calendar:create_onboarding_events",
      "docs:send_template"
    ],
    "action_blocklist": [
      "hris:read_salary",
      "hris:read_ssn",
      "hris:read_bank_info",
      "hris:delete",
      "email:send_freeform"
    ],
    "no_pii_export": true,
    "required": "employee_id",
    "read_only": false
  }
}
```

## 5. Expense Report Agent

**Use case:** An AI that reads receipts and expense submissions, creates records in your expense system, and prepares reports — without the ability to approve, execute, or reject payments.

| Constraint | Value | Why |
|-----------|-------|-----|
| `action_allowlist` | `expenses:read, expenses:create, receipts:read` | Create records, not payments |
| `action_blocklist` | `payments:*, expenses:approve, expenses:reject` | No payment execution |
| `approval_threshold` | `$500.00` | Human reviews large expense reports |
| `no_pii_export` | `true` | Receipts may contain PII |

```json
{
  "template": "expense-report-agent",
  "constraints": {
    "action_allowlist": [
      "expenses:read",
      "expenses:create",
      "expenses:update_draft",
      "receipts:read",
      "receipts:ocr",
      "reports:create"
    ],
    "action_blocklist": [
      "payments:send",
      "payments:approve",
      "expenses:approve",
      "expenses:reject",
      "expenses:delete"
    ],
    "approval_threshold": "$500.00",
    "no_pii_export": true,
    "required": "submitter_id"
  }
}
```

## 6. Meeting Scheduler

**Use case:** An AI that reads calendars, proposes meeting times, creates calendar events, and sends invitations — with limits on external invitations to prevent calendar spam or data leakage.

| Constraint | Value | Why |
|-----------|-------|-----|
| `action_allowlist` | `calendar:read, calendar:create, calendar:invite` | R/W with invite cap |
| `action_blocklist` | `calendar:delete_all, calendar:export_all` | No bulk operations |
| `max_per_day` | `20 external invites` | Prevent calendar bombing |
| `domain_allowlist` | Company domains | External invites to approved domains only |

```json
{
  "template": "meeting-scheduler",
  "constraints": {
    "action_allowlist": [
      "calendar:read",
      "calendar:create",
      "calendar:update",
      "calendar:invite_internal",
      "calendar:invite_external",
      "calendar:find_free_time"
    ],
    "action_blocklist": [
      "calendar:delete",
      "calendar:cancel_all",
      "calendar:export_all"
    ],
    "max_per_hour": 20,
    "domain_allowlist": "mycompany.com,partnercompany.com",
    "required": "organizer_id"
  }
}
```

## Applying Business Templates

```bash
# Apply the Customer Support Agent template
curl -X POST https://uniplex.ai/api/gates \
  -H "Authorization: Bearer uni_xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Customer Support Gate",
    "profile": "L2",
    "trusted_issuers": ["issuer:acmecorp.ai"],
    "template": "customer-support-agent"
  }'
```
