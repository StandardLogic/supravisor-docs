# Developer Tool Templates

Six templates for AI agents in developer workflows. These are typically read-heavy with carefully scoped write permissions — read the codebase freely, but only write to specific, safe targets.

---

## 1. Code Review Agent

**Use case:** An AI that reads pull requests, analyzes code changes, and posts review comments — without the ability to push code, merge branches, or modify repository settings.

| Constraint | Value | Why |
|-----------|-------|-----|
| `action_allowlist` | `git:read, git:read_diff, pr:comment, pr:review` | Read + comment only |
| `action_blocklist` | `git:push, git:merge, git:delete_branch, git:force_push` | No code changes |
| `domain_allowlist` | `github.com, gitlab.com` | Repo platforms only |
| `read_only` | Partial | Write to PR comments, not code |

```json
{
  "template": "code-review-agent",
  "constraints": {
    "action_allowlist": [
      "git:read",
      "git:read_diff",
      "git:read_commits",
      "pr:comment",
      "pr:review",
      "pr:request_changes",
      "pr:approve"
    ],
    "action_blocklist": [
      "git:push",
      "git:merge",
      "git:force_push",
      "git:delete_branch",
      "git:delete_repo",
      "pr:close",
      "pr:delete"
    ],
    "domain_allowlist": "github.com,gitlab.com,bitbucket.org",
    "required": "pr_id",
    "max_per_hour": 30
  }
}
```

<Tip>
  Add your internal GitLab or GitHub Enterprise domain to the `domain_allowlist` if you use a self-hosted instance.
</Tip>

## 2. CI/CD Agent

**Use case:** An AI that monitors CI/CD pipelines, triggers test runs, reads build logs, and reports on failures — without the ability to modify pipeline configurations or deployment targets.

| Constraint | Value | Why |
|-----------|-------|-----|
| `action_allowlist` | `ci:read, ci:trigger_test, ci:read_logs` | Trigger tests, not deploys |
| `action_blocklist` | `ci:deploy_prod, ci:modify_config, ci:delete_pipeline` | Hard blocks |
| `domain_allowlist` | CI/CD platforms | Scoped to known services |

```json
{
  "template": "cicd-agent",
  "constraints": {
    "action_allowlist": [
      "ci:read_pipeline",
      "ci:read_logs",
      "ci:trigger_test",
      "ci:retry_failed",
      "ci:read_artifacts"
    ],
    "action_blocklist": [
      "ci:deploy_production",
      "ci:deploy_staging",
      "ci:modify_config",
      "ci:delete_pipeline",
      "ci:update_secrets"
    ],
    "domain_allowlist": "github.com,gitlab.com,circleci.com,buildkite.com,jenkins.internal",
    "max_per_hour": 20,
    "required": "pipeline_id"
  }
}
```

## 3. Documentation Generator

**Use case:** An AI that reads source code and generates or updates documentation — but cannot modify the code itself. Write access to docs directories only.

| Constraint | Value | Why |
|-----------|-------|-----|
| `action_allowlist` | `files:read, docs:read, docs:write` | Code read, docs write |
| `action_blocklist` | `code:write, files:delete, git:push_code` | No code changes |
| `domain_allowlist` | Internal systems + docs platforms | Scoped access |

```json
{
  "template": "documentation-generator",
  "constraints": {
    "action_allowlist": [
      "files:read",
      "code:read",
      "docs:read",
      "docs:write",
      "docs:create",
      "docs:update"
    ],
    "action_blocklist": [
      "code:write",
      "files:delete",
      "git:push",
      "docs:delete",
      "docs:publish_external"
    ],
    "max_per_hour": 50,
    "read_only": false
  }
}
```

## 4. Bug Triage Agent

**Use case:** An AI that reads bug reports, adds labels, assigns priorities, and links related issues — without the ability to close issues, delete them, or mark them as "won't fix" without human review.

| Constraint | Value | Why |
|-----------|-------|-----|
| `action_allowlist` | `issues:read, issues:label, issues:assign, issues:comment` | Read + metadata |
| `action_blocklist` | `issues:close, issues:delete, issues:lock` | No closing/deletion |
| `required` | `issue_id` | Every action tagged to an issue |

```json
{
  "template": "bug-triage-agent",
  "constraints": {
    "action_allowlist": [
      "issues:read",
      "issues:label",
      "issues:assign",
      "issues:comment",
      "issues:link_related",
      "issues:set_priority",
      "issues:search"
    ],
    "action_blocklist": [
      "issues:close",
      "issues:delete",
      "issues:lock",
      "issues:mark_duplicate",
      "issues:bulk_close"
    ],
    "required": "issue_id",
    "domain_allowlist": "github.com,gitlab.com,linear.app,jira.mycompany.com",
    "max_per_hour": 50
  }
}
```

## 5. Security Scanner

**Use case:** An AI that performs read-only security analysis of a codebase — finding CVEs in dependencies, flagging hardcoded secrets, identifying insecure patterns — without making any automatic fixes.

| Constraint | Value | Why |
|-----------|-------|-----|
| `read_only` | `true` | Absolute — no auto-fixes |
| `action_allowlist` | `files:read, deps:read, git:read` | Read-only analysis |
| `no_pii_export` | `true` | No credential export |
| `domain_allowlist` | Security databases only | CVE lookups only |

```json
{
  "template": "security-scanner",
  "constraints": {
    "read_only": true,
    "action_allowlist": [
      "files:read",
      "code:read",
      "deps:read",
      "git:read",
      "vulnerabilities:lookup"
    ],
    "action_blocklist": [
      "files:write",
      "code:patch",
      "deps:update",
      "git:push",
      "deps:auto_fix"
    ],
    "domain_allowlist": "nvd.nist.gov,osv.dev,snyk.io,cve.mitre.org",
    "no_pii_export": true
  }
}
```

<Warning>
  Even in read-only mode, a security scanner agent will access potentially sensitive code. Ensure the passport's permissions are scoped to specific repository paths, not the entire file system.
</Warning>

## 6. Dependency Auditor

**Use case:** An AI that runs `npm audit` / `pip audit`, reads the output, and generates a report with severity and remediation steps — without updating any packages or modifying lock files.

| Constraint | Value | Why |
|-----------|-------|-----|
| `read_only` | `true` | Audit only, no updates |
| `action_allowlist` | `deps:audit, deps:read, files:read` | Audit commands + read |
| `action_blocklist` | `deps:update, deps:install, deps:remove` | No package changes |

```json
{
  "template": "dependency-auditor",
  "constraints": {
    "read_only": true,
    "action_allowlist": [
      "deps:audit",
      "deps:read_lockfile",
      "deps:read_manifest",
      "files:read",
      "vulnerabilities:lookup"
    ],
    "action_blocklist": [
      "deps:install",
      "deps:update",
      "deps:remove",
      "deps:patch",
      "files:write"
    ],
    "domain_allowlist": "registry.npmjs.org,pypi.org,osv.dev,nvd.nist.gov",
    "max_per_hour": 20
  }
}
```

## Applying Developer Templates

```bash
# Code Review Agent
curl -X POST https://uniplex.ai/api/gates \
  -H "Authorization: Bearer uni_xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Code Review Gate",
    "profile": "L2",
    "trusted_issuers": ["issuer:engineering.mycompany.ai"],
    "template": "code-review-agent"
  }'
```
