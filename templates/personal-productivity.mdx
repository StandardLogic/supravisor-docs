# Personal Productivity Templates

Six templates designed for personal AI assistants. Conservative defaults with sensible daily limits â€” easy to loosen once you've validated the agent's behavior.

---

## 1. Personal Email Agent

**Use case:** An AI assistant that manages your inbox â€” reads, drafts replies, and sends emails â€” with guardrails to prevent accidental mass sends or outreach to new contacts without your approval.

| Constraint | Value | Why |
|-----------|-------|-----|
| `action_allowlist` | `email:read, email:send, email:search` | No delete or export |
| `max_per_day` | `50 actions` | Prevents runaway sending |
| `approval_threshold` | New recipient | Flags emails to unknown contacts |
| `domain_blocklist` | _(none by default)_ | Customize to your needs |
| `read_only` | `false` | Must be able to send |
| `no_pii_export` | `true` | Prevent bulk contact export |

```json
{
  "template": "personal-email-agent",
  "constraints": {
    "action_allowlist": ["email:read", "email:send", "email:search", "email:draft"],
    "max_per_hour": 20,
    "max_per_day": "$0",
    "no_pii_export": true,
    "read_only": false,
    "required": "user_context",
    "action_blocklist": ["email:delete", "email:export", "email:forward_all"]
  }
}
```

<Tip>
  Set `approval_threshold` to require human approval for any email to a recipient not in your address book. Configure this by adding a custom contact-check integration.
</Tip>

## 2. Calendar Manager

**Use case:** An AI assistant that schedules meetings, creates events, and manages your calendar â€” without the ability to delete events or make changes you can't undo.

| Constraint | Value | Why |
|-----------|-------|-----|
| `action_allowlist` | `calendar:read, calendar:create, calendar:update` | Explicit no-delete |
| `action_blocklist` | `calendar:delete, calendar:cancel` | Prevent irreversible changes |
| `max_per_hour` | `20` | Prevent calendar flooding |
| `required` | `event_duration` | Every event must have a duration |

```json
{
  "template": "calendar-manager",
  "constraints": {
    "action_allowlist": [
      "calendar:read",
      "calendar:create",
      "calendar:update",
      "calendar:search",
      "calendar:suggest_times"
    ],
    "action_blocklist": ["calendar:delete", "calendar:cancel_all"],
    "max_per_hour": 20,
    "max_per_day": "$0",
    "required": "event_title",
    "read_only": false
  }
}
```

## 3. File Organizer

**Use case:** An AI that sorts, renames, and moves files in designated folders. Cannot delete files, cannot share files externally, and operates only within scoped directories.

| Constraint | Value | Why |
|-----------|-------|-----|
| `action_allowlist` | `files:read, files:move, files:rename, files:create` | No delete |
| `action_blocklist` | `files:delete, files:share_external, files:upload_external` | Hard blocks |
| `domain_blocklist` | External cloud services | No Dropbox/Drive sync |
| `read_only` | `false` | Must be able to move/rename |

```json
{
  "template": "file-organizer",
  "constraints": {
    "action_allowlist": [
      "files:read",
      "files:list",
      "files:move",
      "files:rename",
      "files:create_folder"
    ],
    "action_blocklist": [
      "files:delete",
      "files:share",
      "files:upload_external",
      "files:download_external"
    ],
    "domain_blocklist": "dropbox.com,drive.google.com,onedrive.live.com",
    "read_only": false,
    "no_pii_export": true
  }
}
```

<Warning>
  Scope the agent's file system access to specific directories at the OS level as well. Supravisor prevents the action â€” your OS permissions are the second line of defense.
</Warning>

## 5. Task Manager

**Use case:** An AI that creates and updates tasks in your task manager. Never deletes tasks (archive instead), and can generate a daily summary to your email.

| Constraint | Value | Why |
|-----------|-------|-----|
| `action_allowlist` | `tasks:read, tasks:create, tasks:update, tasks:complete` | No delete |
| `action_blocklist` | `tasks:delete, tasks:bulk_delete` | Hard blocks |
| `max_per_day` | `200 actions` | Prevent task flooding |
| `required` | `task_title` | Every task must have a title |

```json
{
  "template": "task-manager",
  "constraints": {
    "action_allowlist": [
      "tasks:read",
      "tasks:create",
      "tasks:update",
      "tasks:complete",
      "tasks:archive",
      "tasks:assign",
      "tasks:search"
    ],
    "action_blocklist": ["tasks:delete", "tasks:bulk_delete", "tasks:purge"],
    "max_per_hour": 50,
    "required": "task_title",
    "read_only": false
  }
}
```

## 6. Personal Shopper

**Use case:** An AI that searches for products, compares prices, and can make purchases â€” with hard spending caps and a human approval threshold for anything significant.

| Constraint | Value | Why |
|-----------|-------|-----|
| `max_per_action` | `$500.00` | No single purchase over $500 |
| `approval_threshold` | `$100.00` | Human approves anything over $100 |
| `max_per_day` | `$500.00` | Daily cap |
| `action_allowlist` | `shopping:search, shopping:compare, shopping:purchase` | Scoped actions |
| `domain_allowlist` | Approved retailers | Prevent unknown merchants |

```json
{
  "template": "personal-shopper",
  "constraints": {
    "action_allowlist": [
      "shopping:search",
      "shopping:compare",
      "shopping:read_reviews",
      "shopping:add_to_cart",
      "shopping:purchase"
    ],
    "max_per_action": "$500.00",
    "approval_threshold": "$100.00",
    "max_per_day": "$500.00",
    "max_cumulative": "$2000.00",
    "cumulative_window": "30d",
    "domain_allowlist": "amazon.com,bestbuy.com,target.com,walmart.com",
    "no_pii_export": true
  }
}
```

<Warning>
  For any template that involves payments, always configure `approval_threshold` and test the approval flow before deploying in production.
</Warning>
